# Worcester County House Price Prediction

A machine learning project that predicts single-family home sale prices in Worcester County, MA (primarily Harvard, MA). Includes a full analysis notebook, a reusable data pipeline that fetches live data from Redfin, and an interactive Streamlit web app for price predictions.

## Project Structure

```
├── harvard_house_market_analysis.ipynb   # Full analysis notebook (EDA, modeling, evaluation)
├── data_pipeline.py                      # Redfin data fetching, cleaning & caching module
├── app.py                                # Streamlit prediction web app
├── house_data.csv                        # Static fallback dataset
├── model/
│   └── best_model.pkl                    # Trained model + metadata (generated by notebook)
├── data/
│   └── redfin_data.csv                   # Cached Redfin data (auto-generated, gitignored)
├── requirements.txt                      # Python dependencies
└── .gitignore
```

## Key Results

- **Dataset:** ~77 unique property sales fetched live from Redfin (Harvard, Devens, Ayer)
- **Best Model:** Ridge Regression (selected via Leave-One-Out Cross-Validation)
- **Top Price Drivers:** Square footage, lot size, and bathroom count
- **Interactive App:** Users input property features and get an instant price prediction

## Notebook Sections

1. **Setup** — imports, config, helper functions
2. **Data Loading & Audit** — live Redfin fetch with fallback, deduplication, missingness analysis
3. **EDA** — price distributions, correlation heatmap, scatter plots, geographic visualization, temporal trends
4. **Feature Engineering** — house age, cyclical month encoding, derived ratios
5. **Modeling** — Linear Regression, Ridge, Random Forest, XGBoost with temporal train/test split
6. **Evaluation** — comparison table, actual vs predicted, residual diagnostics, SHAP analysis
7. **Time Series Trends** — monthly median price with trend line
8. **Model Export** — saves best pipeline to `model/best_model.pkl`
9. **Conclusions** — findings, limitations, recommendations

## Data Pipeline

`data_pipeline.py` fetches sold-home data from Redfin's Stingray CSV endpoint:

- **Live fetch** with pagination and rate limiting
- **Local caching** (refreshes after 7 days)
- **Automatic fallback** to `house_data.csv` if the network fetch fails
- **Cleaning**: snake_case columns, type coercion, deduplication, sparse column removal

```python
from data_pipeline import load_data
df = load_data()  # fetches from Redfin or loads cache/fallback
```

## Streamlit App

An interactive web app where users input house features and get a predicted sale price.

**[Try the live app here](https://mahousepriceprediction.streamlit.app/)**

**Features:**
- Sidebar with sliders/selectors for square feet, beds, baths, lot size, year built, city, and sale month
- Large predicted price card with confidence interval (model MAE)
- Visual comparison to dataset price distribution
- Refresh button to pull latest Redfin data

### Run Locally

```bash
streamlit run app.py
```

## Limitations

- Small sample size (~54–77 sales depending on data freshness)
- Geographic concentration — mostly Harvard, MA
- Short time span (~11 months) — insufficient for seasonal/macro trend modeling
- No interior quality features (finish, renovations, school ratings)

## Technologies

Python, pandas, scikit-learn, XGBoost, SHAP, Matplotlib, Seaborn, Streamlit, Redfin Stingray API
